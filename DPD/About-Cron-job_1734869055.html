1. What is Cron job?

A Cron job is a scheduled task in Unix-like operating systems that runs automatically at specified intervals or times.

It's managed by the cron daemon, which reads configuration files called "crontabs" to determine what commands or scripts to execute and when.

Each cron job is defined using a specific time syntax composed of five fields (minute, hour, day of month, month, day of week), followed by the command to run.

For example, [0 2 * * 1] means "run at 2:00 AM every Monday."

2. How to set a Cron job?

  • Prepare your Shell script file
    : Create a shell script file containing the commands you want to run at regular intervals.

If you have only a few commands to run, you can write them directly in the Cron configuration without creating a separate script file.

  • Open Cron job configuration file

    sudo crontab -e

  • Set the regular intervals time to run a script file

Each digit refers: [Min] [Time] [Day] [Month] [Day of week]

  • Check currently running Cron jobs

    sudo crontab -l
