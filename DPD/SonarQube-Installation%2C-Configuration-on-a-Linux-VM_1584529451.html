Created by Jindo Kim, last modified by Chris Park on Feb 04, 2025

Objective

While the linked guide (https://varlab-dev.atlassian.net/wiki/x/CICbVw ) outlines the installation process on an Azure Windows VM using Docker, this document focuses on a Linux platform with a non-Docker installation approach.

1. Installation

  • Most installation steps, including the PostgreSQL installation, have been completed as described at the following link: https://hostnextra.com/learn/tutorials/how-to-install-sonarqube-on-ubuntu. Steps 1 to 6 from the guide are sufficient for this setup.

The NoClassDefFoundError is a known issue in versions of SonarQube below 10.6. Since the instruction link specifies version 10.5.1, it would be advisable to refer to the official documentation and install the latest version.

  • Details about the issue can be found in the Jira ticket ( CORE-1440 - Getting issue details... STATUS )

2. PostgreSQL

2.1 PostgreSQL Management Configuration

  • It is hosted under the system's postgres user account on Linux, with SonarQube connecting to the sonar database using the sonar user account for storing and retrieving data.

    • The sonar user account and password are used in the SonarQube properties file to configure database connectivity.

3. SonarQube

3.1 SonarQube Management Configuration

  • SonarQube is hosted under the system's sonarqube user/group account, ensuring proper isolation and security (as defined in its service configuration).

3.2 Nginx Configuration for Hosting SonarQube

  • For more details, please refer to the link: https://varlab-dev.atlassian.net/wiki/spaces/DC/pages/1584005153/System-level+security+configuration+on+Linux+Jenkins+VM#2.2.1-Nginx-Configuration

3.3 Integration of SonarQube in Jenkins

  • For more details, please refer to the link: https://varlab-dev.atlassian.net/wiki/spaces/DPD/pages/1469808648/Static+Code+Analysis+using+SonarQube#2.3-Install-Sonar-Scanner-Plugin-in-Jenkins

SonarQube Auth Tokens Update: Project Token Deprecated, User Token Adopted

  • Project Token: No longer required for the Jenkins pipeline, as authentication now relies on the User Authentication Token.

3.4.5 The “waitForQualityGate” prints “IN PROGRESS” and waits forever

  • To execute SonarQube analysis and retrieve the quality gate status, the process follows these steps:

    1. withSonarQubeEnv: Runs the SonarQube analysis.

    2. waitForQualityGate: Waits for the analysis to complete and retrieves the quality gate status.

    3. For more details, please refer to this link: https://www.jenkins.io/doc/pipeline/steps/sonar/#sonarqube-scanner-for-jenkins

  • For an unknown reason, the waitForQualityGate function retrieves the status before it updates from ‘IN_PROGRESS’ to either ‘SUCCESS’ or ‘FAILURE’.

  • This is a well-known issue among the SonarQube Community. For details, please refer to this link: https://community.sonarsource.com/t/need-a-sleep-between-withsonarqubeenv-and-waitforqualitygate-or-it-spins-in-in-progress/2265

Jenkins Log / Screenshot

  • For more details, please refer to this Jenkins Log link: https://jenkins.vconestoga.com/job/Asset-Store/job/Asset-Store-Pull-Request/100/console

Checking status of SonarQube task '93250e12-4bd0-410a-991a-7a4b8d3ce97c' on server 'SonarServer'
SonarQube task '93250e12-4bd0-410a-991a-7a4b8d3ce97c' status is 'IN_PROGRESS'
Aborted by admin

  • The deprecated the waitForQuailtyGate Function

// Set a timeout for waiting on the Quality Gate result
  int timeoutDuration = 10
  String timeoutUnit = 'MINUTES'

  timeout(time: timeoutDuration, unit: timeoutUnit) {
    catchError(buildResult: 'SUCCESS', stageResult: 'FAILURE') {
    // Wait for SonarQube Quality Gate results
    def qualityGate = waitForQualityGate()
    echo "Quality Gate Status: ${qualityGate.status}"

    if (qualityGate.status != 'OK') {
    error "SonarQube Quality Gate failed"
    }
  }
}

  • Applied Solution

    • No longer to use the waitForQuailtyGate built-in function

    • Create our own SonaQube API call function checkQualityGateStatus() and replace waitForQuailtyGate() function.
