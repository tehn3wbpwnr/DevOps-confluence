Summary

  • This document provides a guide on enabling WebGL builds after installing Unity Hub and the Unity Editor on Ubuntu via a graphical user interface (GUI) accessed through RDP. It also includes step-by-step instructions for installing and utilizing Git Large File Storage (LFS) and FFmpeg (Fast Forward Moving Picture Experts Group).

1. Environment Preparation

1.1 Install Git Large File Storage

# Update the package list to get the latest version of the packages.
sudo apt update

# Install Git Large File Storage (LFS). Git LFS is an extension for managing large files.
sudo apt install git-lfs

# Install Git LFS for the current user environment. This command sets up Git LFS to be used.
git lfs install

# Pull large files from the remote repository using Git LFS. This command downloads files managed by LFS.
git lfs pull

  • Installing and initializing Git Large File Storage on a headless Ubuntu VM requires additional configuration compared to a Windows environment. Using the git lfs pull command within the project folder, an environment that temporarily supports proper WebGL builds has been established, even though this is not an ideal approach for leveraging Git LFS functionality. Further research and adjustments seem necessary to optimize the use of Git Large File Storage.

Important:

To avoid potential disruptions in the WebGL build process, enable Git Large File Storage (LFS) in the project configuration.

1.2 Install FFmpeg (Fast Forward Moving Picture Experts Group)

# Install ffmpeg
sudo apt-get install ffmpeg
About FFmpeg

FFmpeg is a free and open-source software project consisting of libraries and programs for handling video, audio, and other multimedia files and streams.

https://ffmpeg.org/

  • In a headless Ubuntu/Linux environment, when performing Unity WebGL builds in batch mode, the absence of FFmpeg results in the WebGL build being generated without audio. This behavior is documented in the Unity 6 Documentation , which can be found here. However, please note that this information may change in future updates or with newer Unity versions as the platform evolves.

There is an issue where the build process does not fail as expected when FFmpeg is missing and audio conversion fails. Instead, the build is reported as successful, even though the resulting build lacks audio. To reproduce the audio conversion process or verify logs, it is necessary to completely delete the project and restart the process from a fresh git clone. Unity will only attempt audio conversion again during the WebGL build, regardless of whether FFmpeg is present.

2. WebGL Builds

2.1 Command:

  • xvfb-run -a

  • <Editor Path>

  • -batchmode

  • -projectPath "<Project Path>"

  • -buildTarget WebGL

  • -logFile "<Log File Path>"

  • -executeMethod <Build Method>

  • -quit

  • For example, when applied to the respiratory therapy DLX project:

xvfb-run -a \
/home/gui/Unity/Hub/Editor/6000.0.12f1/Editor/Unity \
-batchmode \
-projectPath "/home/gui/Desktop/respiratory-therapy" \
-buildTarget WebGL \
-logFile "/home/gui/Desktop/respiratory-therapy/reports/webgl.log" \
-executeMethod Builder.BuildWebGL \
-quit
