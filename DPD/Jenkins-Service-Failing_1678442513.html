DevOps / Pipeline & Deployment : Jenkins Service Failing

Created by Chris Park, last modified on Mar 10, 2025

: This issue occurred after the jenkins.service file was overwritten by 'apt upgrade'.

Contents

  • 1. Diagnosis the issue
  • 2. Troubleshooting

1. Diagnosis the issue

The situation is like the screenshot shown as below

For a more detailed diagnosis, please run the command 'systemctl show jenkins | grep -E '^User=|^Group=' to check the system environment variables of the Jenkins service. The output should resemble the log below.

User=jenkins-ubuntu-dev-test
Group=jenkins-ubuntu-dev-test

If the outputs are 

User=jenkins
Group=jenkins

or does not match the VM host user, we need to update the jenkins.service file.

2. Troubleshooting

Here is the break downs for fixing this issue.

  1. Navigate to the service file:
    Go to /lib/systemd/system/ and open the jenkins.service file with superuser permissions.

  2. Navigate to the service file:
    Go to /lib/systemd/system/ and open the jenkins.service file with superuser permissions.

  3. Update the user and group:
    Change these values to match the VM host user (e.g., jenkins-ubuntu-dev-test).

  4. Enable system logging:
    Uncomment the following line to enable system logging
    >> ‚Äò#Environment="JENKINS_LOG=%L/jenkins/jenkins.log‚Äô for system logging

  5. Save the changes and exit the editor

  6. Reload the systemd daemon:
    >> sudo systemctl daemon-reload

  7. Restart the Jenkins service:
    >>sudo systemctl restart jenkins

  8. Check the service status:
    >> sudo systemctl status jenkins
