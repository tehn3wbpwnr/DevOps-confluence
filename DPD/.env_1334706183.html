DevOps / Pipeline & Deployment : .env

Created by Chris Park, last modified on Sept 13, 2024

This documentation explains each environment variable’s usage and role inside of the LTIjs service.

  • 1. What is a .env file?
  • 2. Where is a .env file?
  • 3. What kind of environment variables do we need?

1. What is a .env file?

: . env file is a file that stores the environment variables and their corresponding values ​​required to host the LTIjs service.

2. Where is a .env file?

: This ‘.env' file is located it the main directory of LTIjs.

Example screenshot of dlx-webhost

If you can’t find it, make sure to use “ls -a“ to see all the hidden file.

3. What kind of environment variables do we need?

  1. DB_URL
    : MongoDB connection URL. It should include the listening port number and login information with the belonging database table.

  2. URL
    : It should be the Server URL. It will be used when the LTIjs service passes all the redirect URLs to LMS. So, it is wrong, LMS will redirect to the wrong path.

  3. REG_NAME
    : The name of an external tool when it shows up after the registration.

  4. LTI_KEY
    : Key used to sign cookies and tokens. It can be a random string, but we need something.

  5. LOG_DIR
    : The directory path for logging.

  6. PORT
    : Listening port of LTIjs service.
