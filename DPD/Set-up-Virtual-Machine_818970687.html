DevOps / Pipeline & Deployment : Set up Virtual Machine

Created by Jonathan Bezeau, last modified on Apr 11, 2024

Process

  • log in to Azure portal

  • create new Virtual Machine in VARLab subscription / BuildSystem resource group

    • 8+ CPU server shape

    • Windows OS

    • Public IP

    • no extra disks

    • accept defaults for networking

    • allow inbound traffic on port 80

  • click on created resource (VM)

    • navigate to Networking

    • Validate that there is a rule for inbound HTTP requests on port 80

    • Create the inbound rule if there isn’t one

  • At VM Overview click “Connect” and follow steps to download .rdp file and attach using Windows Remote Desktop Protocol

    • on the Remote Desktop search in the task bar for Windows Firewall and open the application

    • Click “advanced settings”

    • Click “inbound rules” and “new rule…”

    • allow inbound traffic on port 80
