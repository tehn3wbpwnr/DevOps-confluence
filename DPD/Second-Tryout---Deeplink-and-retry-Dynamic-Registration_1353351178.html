Summary

  • In the first tryout, we successfully completed the steps of receiving the Registration (Client_id) and Deployment (Deployment_Id), as well as creating a link for the Basic Launch Type. In this second tryout, we are focusing on attempting the Create Link for the Deeplink Type (a method that allows connecting multiple DLXs with a single link) and retrying Dynamic Registration. This document covers the analysis based on application and HTTP logs during the second tryout, and compares them with Moodle logs from another platform under the same conditions, aiming to serve as a valuable resource for the next tryout.

Comparison and Analysis

  • Target Link URI:

    • eConestoga: specific URI - https://webdlx.vconestoga.com/deeplink

    • Moodle: Uses a more general URI - https://webdlx.vconestoga.com

  • Additional Parameters:

    • eConestoga:

      • lti_storage_target: log from Moodle doesn’t contains

        • To gain further insights into this parameter, please refer to this link: https://www.imsglobal.org/spec/lti-cs-oidc/v0p1

      • lti_message_hint: providing more context or hints then Moodle

    • Moodle:

      • lti_message_hint: it contains only simple value, '0'.

  • Cookies and Security (This Analysis is not yet conclusive) :

    • eConestoga: Sends a cookie in the headers

    • Moodle: Not provided
