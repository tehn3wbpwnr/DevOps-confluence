DevOps / Pipeline & Deployment : Set Up Jenkins Build System

Created by Jonathan Bezeau on Oct 03, 2023

Overview

VARLab is developing build infrastructure to organize and automate building DLX projects.

Components

Our setup will involve our developers, Unity game development engine, Bitbucket repository service and a Jenkins build automation server running on a Windows VM either in the cloud or on campus.

Process

The ultimate process will look like this:

  • Developer starts a git branch for their ticket, and develops their solution locally

  • Developer commits changes and pushes to the Bitbucket repo

  • Developer submits Pull Request to Bitbucket

    • Bitbucket sees the PR and triggers a web hook on the Jenkins build server

    • Jenkins clones the developer’s branch

    • Jenkins merges the latest changes from main

    • Jenkins runs NUnit tests on the developer’s merged branch

    • if all NUnit tests are successful

      • Jenkins builds the developer’s merged branch

      • Jenkins packages[1] and deploys[2] the executable output

    • Jenkins messages[3] the developer and updates the PR with the build status

  • DnA team member approves PR based on code review and successful build output

  • Developer merges code to main (Jenkins could merge code to main on successful build)
