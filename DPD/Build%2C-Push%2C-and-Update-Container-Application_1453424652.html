Access Setup

  • create a Managed Identity for the build VM (jenkins-win-docker)

  • assign acr push to the jenkins-win-docker identity on the container registry (WebBuilds)

  • assign contributor to the jenkins-win-docker identity on the container app resource group (AssetStore)

    • this could be more granular but we couldn’t identify a Job Role for pushing to container apps, and chose not to manage separate access rights for all container app instances

Build And Deploy

Use PowerShell to run commands using the az Azure CLI

Login Using Manage Identity

: az login --identity --username xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

Login to Container Registry

: az acr login --name  WebBuilds

Build The docker file

: docker build -t http://webbuilds.azurecr.io/assetstore-server:1.2 C:/ProgramData/Jenkins/.jenkins/workspace/Asset-Store/Asset-Store-Deployment-Pipeline/JS_Project/server/.

Push The Image to The Container Registry

: docker push http://webbuilds.azurecr.io/assetstore-server:1.2

Update The Container App

: az containerapp update --name assetstore-server --resource-group AssetStore --image http://webbuilds.azurecr.io/assetstore-server:1.2

  • --name must match the name of the container already set up on an application
