DevOps / Pipeline & Deployment : Create Bitbucket Report

Created by Chris Park, last modified on Apr 17, 2025

This documentation explains how to generate a Bitbucket report on the pull request page. The report format varies depending on the pipeline type and the corresponding stage in the Jenkins build.

Please remind, reports are only generated by PR pipelines. NOT deployment pipelines.

If you're not sure what Bitbucket reports look like, please refer to this documentation!

  • DLX Bitbucket Reports: https://varlab-dev.atlassian.net/wiki/x/BYBQZ

  • JS Bitbucket Reports : Web Development Bitbucket Reports

Jenkins is using two API calls to create a Bitbucket report on PR page.

  1. PUT: Create or update a report

  2. PUT: Create or update an annotation

Please check the API documentations for furthermore details!

3. How Jenkins send the Bitbucket Report?

: "To submit a Bitbucket report on the PR page, Jenkins makes two API calls. First, it sends an HTTP PUT request to Bitbucket Cloud to create the report. Then, it sends another HTTP PUT request to attach annotations to the created report

It is very important using a consistent report URL when we put the annotations on the report in terms of better management.

  • Report URL:
    {pr_repo}/commit/{commit_hash}/reports/{report-name}

  • Annotation URL:
    {pr_repo}/commit/{commit_hash}/reports/{report-name}/annotations/{ticket_number}-{index}

3.1 Path Parameters

: These are the parameters that must be included in the request URL. Each parameter is explained in the table below.

Parameter Name

Description

Example

commit

The full commit hash for which Jenkins sends the build status.

This commit hash is tracked by Jenkins in env.COMMIT_HASH

 

workspace

This refers the name of workspace which repository belongs to

Jenkins uses env.JOB_REPOwhich is the combination of workspace/repo_slug.

 

repo_slug

This refers the name of repository.

 

reportID

A unique and meaningful numeric or string ID

Matching with the actual report name is suggested

 

Example URL

https://api.bitbucket.org/2.0/repositories/{workspace}/{repo_slug}/commit/{commit}/reports/{reportId}

3.2 Header

: This header configuration ensures that Bitbucket API requests are sent and received in JSON format, with the required Bitbucket Access Token (stored in env.BITBUCKET_ACCESS_TOKEN) included for authorization.

It sets "Accept" and "Content-Type" to "application/json" and prepends "Bearer " to the token in the "Authorization" field.

Please be cautious of ‘WHITE SPACE' after the 'Bearer '

Header

Accept

application/json

Content-Type

application/json

Authorization

Bitbucket Access Token is required for this field.

Bitbucket Access Token is stored by Jenkins as an env.BITBUCKET_ACCESS_TOKEN

Example Header

headers = {
    "Accept": "application/json",
    "Content-Type": "application/json",
    "Authorization": "Bearer " + access_token
}

3.3 Request Body (application/json)

: This content includes fields only that are using in our pipeline. Please review the Bitbucket Cloud API if you want to check all possible fields in this PUT: Update a build status for a commit.

Parameter Name

Description

Example

title

The title displayed for the report on the PR page

details

A short description explaining the purpose of the report.

report_type

Categorizes the report as one of…

  • Valid Values:
    [SECURITY, COVERAGE, TEST, BUG]

reporter

Name of the tool or organization that generated the report.

result

The outcome of the report.

  • Valid Values:
    [PASSED, FAILED, PENDING]

link

URL to view the full report in an external system.

data

The set of data the report needs to be included.
