Simple Definition:

  • Regression testing is simply the process of running existing unit, integration, and end-to-end tests on the codebase after new changes have been merged to ensure that they still pass successfully. This emphasizes that regression testing is not about creating new test cases, but about reusing existing tests to verify that recent changes have not disrupted anything unintentionally.

    • https://www.browserstack.com/guide/retesting-vs-regression-testing

      • Regression Testing:

        • It involves rerunning once-executed test cases to ensure they still pass after the changes are made.

      • Re-testing:

        • The process of testing a previously failed test case after its defects have been fixed.

Possible Scenario:

Run Tests

  • Execute all existing tests from the Main Branch in the New Branch.

Compare Results

  • Compare the results with those from the Main Branch.

    • Success: If all tests pass in both branches, Regression Testing is successful.

    • Failure: If any test fails in the New Branch but passed in the Main Branch, Regression Testing has failed.
