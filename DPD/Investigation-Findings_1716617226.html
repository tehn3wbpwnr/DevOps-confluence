Features In Depth

This application enables record-and-playback functionality, allowing users to interact with the application while the tool captures snapshots of their interactions. When a user selects a button on the UI screen, the tool records an image of that specific element. During testing, game drivers can automatically recognize and select the recorded button. Additionally, the system supports assertions and validations—ensuring that expected UI changes occur. For example, if clicking a button should trigger a new UI appearance, validation can be performed using image recognition to confirm the expected behavior

Second way to automate testing is through POCO that allows for objects(UI components) to be exposed through AirTest IDE

Pros

  • Don’t need to inject code other than the Poco SDK to do testing as all of testing done through AirTest IDE.

  • Can run via command line

  • Updated regularly

  • Can be done via OpenGL online since image recognition allows it to be independent

    • Could serve useful for more user end testing

  • Situations where save and load is not implemented but developers need quick way to get to certain stages of sim quickly.

    • Though do due to nature of image recognition, changes made could easily break tests.

Cons

  • The Poco exposed hierarchy feature did not function correctly when tested with the forked Unity UI Toolkit repository. Input coordinates were consistently misaligned, significantly offset from the actual button positions, effectively rendering the feature unusable.

    Additionally, AirTest recommends using Poco whenever possible to minimize reliance on image recognition, which helps reduce risks. However, this approach is less viable because Poco does not natively support UI Toolkit, and even when using a forked Poco SDK with UI Toolkit support, my testing showed that it still failed to work.

Challenges with Image Recognition

Image recognition can be unreliable, as its accuracy depends on various factors. For instance, if multiple similar icons exist on the screen, the system may incorrectly recognize an icon and pass the test—even if the intended icon is not actually visible.

While certain features enhance image recognition reliability, it is still susceptible to validation failures. If an icon is modified, manual intervention is required to retake a snapshot of the UI component to ensure continued test accuracy.

For more details on improving image recognition accuracy, refer to the documentation linked below.
