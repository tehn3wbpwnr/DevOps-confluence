1. What is Grafana?

: Grafana is an open-source data visualization and monitoring tool that visualizes data such as metrics, logs, and traces from various sources. It provides customizable dashboards to display collected data from the configured data sources.

1.1 Notable characteristics of Grafana

Grafana offers more than just data visualization. Here is a list of services you can use with Grafana:

Alerts

: Through the Grafana API, we can configure alert rules for each job and send SMS messages, emails, or app notifications based on the notifier channels we have defined.

Annotations

: Developers can add meaningful annotations at graphs. These annotations can be very useful for troubleshooting when the system encounters issues.

Plugins

: Grafana has a lot of plugins so that developers can integrate different software or app easily such as Jira, Mongo DB, Azure Cloud, dash board â€¦etc. You can explore more about the Grafana plugin though these link.
- Grafana Dashboard: https://grafana.com/grafana/dashboards/
- Grafana Plugins: https://grafana.com/grafana/plugins/

1.2 Could VS Local

: We can use Grafana Could with free account or install it locally, so we can consider which one is more valuable in our system.

Grafana Cloud

: Grafana Cloud is a fully managed service provided by Grafana Labs. Users can focus on data visualization and monitoring right away without managing servers.

2. What is Prometheus?

: Prometheus is an open-source monitoring and alerting system, primarily used for collecting and storing time-series metrics data. In other words, Prometheus collects metrics data from Jenkins, and Grafana queries this data from Prometheus to visualize it on the dashboard.
