Goal:

  • To encourage the TDD principle by ensuring that unit test cases are developed alongside the code.

    • https://medium.com/@vishwasacharya/node-js-testing-best-practices-and-frameworks-for-reliable-apps-656f1cc51426#:~:text=Test%2DDriven%20Development%20(TDD)%20with%20Node.js

    • https://www.spiceworks.com/tech/devops/articles/what-is-tdd/

Purpose

  • To review whether appropriate unit test cases have been written for the modified or newly added code, allowing the team to prompt or decide on policies regarding the inclusion of unit tests.

Operation

Track Code Changes with Git Diff

  • Use git diff to identify modifications in the codebase.

Extract Method and File Names

  • Extract the names of modified methods and their associated files from the git diff output.

Locate Relevant Test Cases

  • Use the method and file names to identify test cases that adhere to the team's naming conventions.

Extract Method and Test Case Code

  • Extract the full code of the modified methods and their corresponding test cases.

Create HTML or a Similar Tool for Display

  • Develop an HTML page or similar tool to display the method code on the left and the test case code on the right, side by side.

Facilitate Review and Feedback

  • Provide a platform for reviewers and developers to discuss and align on the method and test case code.

Enable Test Execution

  • Optionally include functionality to run tests directly from the HTML interface, allowing for immediate feedback on test cases.

Bitbucket Integration

  • Upon the reviewers approval, a Bitbucket webhook triggers a request to the Jenkins server to advance the CI/CD pipeline to the next stage.
