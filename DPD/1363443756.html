Created by Jindo Kim, last modified on Sept 26, 2024
  • Goal
  • Purpose
  • Operation
  • References / Tools

Goal

  • To verify that small functions or components of the code work independently as expected.

Purpose

  • To test individual modules or components, ensuring that specific code changes maintain the expected behavior of these elements and to detect and resolve bugs or errors early.

Operation

  • Verify that each function or method performs as expected in an isolated environment.

  • Validate that each function or method produces the correct output for given inputs.

  • Test how the functions or methods handle unexpected inputs or error conditions (edge cases)
