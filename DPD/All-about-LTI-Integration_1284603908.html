1. What is LTI(Learning Tools Interoperability)?

: Simply, LTI is a standardized protocol that uses in learning platform(e.g. eConestoga) to communicate securely and smoothly with learning-related external tools(e.g. Zoom, Google Meet, Microsoft Teams …etc.).

Our target deployment platform is eConestoga and eConestoga is following the LTI 1.3/Advantage. Therefore, we also need to follow and support LTI1.3/Advantage to deploy our dlx project to eConestoga.

LTI1.3/Advantage supports OAuth 2.0 and OpenID authentication.
The summary of authentication process between LMS and server looks like the chart below

1.1 OAuth 2.0

: Used to authorize the user to access certain resources. OAuth allows third-party applications to access resources without the user providing their credentials.

- OAuth 2.0 authentication is used on the server side(dlx-webhost) to grant LMS(eConestoga or Moodle) access to specific resources(dlx WebGL).

- OAuth 2.0 authentication is processed in the form of JWT(Json Web Token) called ‘ltik’.

More details about OAuth 2.0 is described at this document OAuth 2.0 and LTI Authorization

1.2 Open ID Connect

: Used to authenticate the user's identity. The user is authenticated through an OpenID provider, and the LMS or external tool uses this authentication result to verify the user's identity.

More details about OIDC is described at this document Open ID Connect and LTI Integration

2. What is LMS(Learning Management System)?

: LMS is learning platform such as eConestoga, D2L, or Moodle. Those learning platform has their own learning tools, but also using tools from outside of LMS and we need to register that 'external tool' to host our dlx projects on eConestoga.

If we have ‘external tool' that provides dlx projects(WebGL Builds), we don’t have to upload the latest dlx project every time because all projects will be always up-to-date through ‘deployment pipeline' and hosted through this 'external tool’.

- It is literally Automated Deployment ! -

So, we need two things to reach out 'Automated Deployment'

  • First we need server that hosts our dlx projects
    >> It will be a external tool
    >> In this case, our dlx-web host is our external tool

  • Second we need to register our external tool on eConestoga
    >> We need permission from eConestoga

3. What dlx-web host server do?

: dlx-web host server is the server to host dlx projects as a external tool of LMS.

In dlx-web host server, a service called ‘LTIjs’ is listening to requests from the LMS side behind the scenes.

This LTIjs service supports LTI 1.3, OAuth 2.0, and Open_ID connection that they are required for LTI intergration with LMS. More detailed things about LTIjs service will be discussed in the link below

4. What is Moodle?

: Moodle is the testing environment of LMS. So, we can test the LTI integration before we actually apply any changes on our target deployment platform which is eConestoga.

Moodle is the best environment to test and investigate about the LTI integration process between LMS and external tool. We can test such as external tool registration, resource hosting, deeplink process, or grading service.

More detailed things such as ‘how to register external tool’ or 'how to submit a grade' will be discussed on the link below.
