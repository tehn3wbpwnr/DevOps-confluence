1. Install Jenkins on Ubuntu

1. Update Software

Run the following commands to update and upgrade system packages:

sudo apt update
sudo apt upgrade

2.Download Jenkins Packages for Debian/Ubuntu

Download and set up the Jenkins package key:

sudo wget -O /usr/share/keyrings/jenkins-keyring.asc \
    https://pkg.jenkins.io/debian-stable/jenkins.io-2023.key

Add the Jenkins repository to your system:

echo "deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc]" \
    https://pkg.jenkins.io/debian-stable binary/ | sudo tee \
    /etc/apt/sources.list.d/jenkins.list > /dev/null

3. Install JDK 17

Update the package list and install JDK 17:

sudo apt-get update
sudo apt-get install fontconfig openjdk-17-jre

Varify that Java is installed and using the correct version:

java -version

Note: Jenkins requires OpenJDK 17 instead of OpenJDK 11 for Debian 12.

4. Install Jenkins

Install Jenkins using the following command:

sudo apt-get install jenkins

5. Verify Jenkins Installation

Check the status of Jenkins to ensure it is running correctly:

systemctl --full status jenkins

Retrieve the initial admin password for Jenkins setup:

sudo cat /var/lib/jenkins/secrets/initialAdminPassword

6. Enable Jenkins to Start at Boot (Optional)

To configure Jenkins to start automatically at system boot:

systemctl enable jenkins

Reference:

Download Jenkins Debian Packages: https://pkg.jenkins.io/debian-stable/

2. Connect with Bitbucket using Webhook

To set up a connection between Jenkins and Bitbucket, you need the following four prerequisites:

  1. Bitbucket App Password

  2. Jenkins Server SSH Key

  3. Bitbucket Repository Webhook URL

  4. Bitbucket Repository Access Token

2.1 Set Up Bitbucket App Password (Deprecated)

App passwords are user-based access tokens for scripting tasks and integrating tools with Bitbucket Cloud.

A. Generate Bitbucket App Password

  1. Log in to Bitbucket with the admin account (ci-catherine) and navigate to Personal Bitbucket settings.

  1. Go to the App password menu and click Create app password.

  1. Fill out the Create app password form with the necessary details.

  1. Save the generated App password. (Note: You will not be able to view the password again)

B. Register Bitbucket App Password in Jenkins

  1. Navigate to Jenkins → Manage Jenkins → Credentials.

  1. Go to Global Credentials and click Add Credentials.

  1. Complete the New credentials form with the App Password.

  • The Username must match Bitbucket username

2.2 Set Up Jenkins Server SSH Key

SSH is used to establish a secure connection between the Linux server and Bitbucket Cloud. This secure connection enables Jenkins to import pipeline scripts from the Bitbucket repository.

A. Generate SSH Key Pair on Linux Virtual Machine

  1. Access the Linux virtual machine and log in as jenkins account:

The jenkins account is the default for the Jenkins server. However, this can differ depending on the configuration.

sudo su - jenkins

  1. Generate SSH key pair using Ed25519 algorithm:

ssh-keygen -t ed25519 -C "varlabcc@gmail.com"

  1. Add Bitbucket to the list of knwon_hosts:

ssh-keyscan -t ed25519 bitbucket.org >> ~/.ssh/known_hosts

  1. Verify the generated key pair:

  • View the public key:

cat ~/.ssh/id_ed25519.pub

  • View the private key

cat ~/.ssh/id_ed25519

B. Register the Public SSH key in Bitbucket

  1. Log in to Bitbucket with admin account (ci-catherine) and navigate to Personal Bitbucket settings.

  1. Go to the SSH keys menu and click Add key.

  1. Fill out the form with label and the public key generated in the previous step.

C. Register the Private SSH key in Jenkins

  1. Navigate to Jenkins → Manage Jenkins → Credentials.

  1. Go to Global Credentials and click Add Credentials.

  1. Complete the New credentials form, ensuring the private key is added.

2.3 Set Up Bitbucket Repository Webhook URL

The Webhook URL facilitates integration between Bitbucket and Jenkins, enabling automated actions upon repository events such as pull request creation, update, and merges).

A. Install the Generic Webhook Trigger Plugin in Jenkins

  1. Navigate to Jenkins → Manage Jenkins → Plugins.

  1. Search for Generic Webhook Trigger and install the plugin.

B. Set Up the Webhook URL for the Target Repository

  1. Log in to Bitbucket with the admin account (ci-catherine) and navigate to the targeted repository (For example, CORE Sandbox).

  2. Go to Repository settings.

  1. Navigate to the Webhooks and click Add webhook.

  1. Complete the form with the Webhook URL in the following format:

  • <JENKINS_URL>/generic-webhook-trigger/invoke?token=<trigger_token>

Note: <trigger_token> can be any descriptive value, and this will be stored in Jenkins credentials.

C. Register the Webhook Token in Jenkins

  1. Navigate to Jenkins → ${Project Folder} → Credentials.

  1. Go to Stores scoped to ${Project} and click Add Credentials.

  1. Complete the New credentials form with the Webhook Token.

2.4 Set Up Bitbucket Repository Access Token

A Repository Access Token allows Jenkins to interact with a specific Bitbucket repository securely.

A. Set Up the Webhook URL for the Target Repository

  1. Navigate to the Repository settings for the target repository.

  1. Go to the Access tokens and click Create Repository Access Token.

  1. Complete the Create Repository Access Token form.

  1. Save the generated Repository Access Token. (Note: You will not be able to view the password again)

B. Register the Repository Access Token in Jenkins

  1. Navigate to Jenkins → Project Folder (e.g., CORE-Sandbox-V2).

  1. Open the Credentials inside the Project Folder.

  1. Go to Global Credentials scoped to the Project Folder.

  1. Click Add Credentials.

  1. Complete the New credentials form with the Access Token.

2.5 Final Configuration for the Project Job in Jenkins

Note: This step focuses only on configuring the Jenkins project job to connect with Bitbucket. Other configurations are not covered in the instructions below.

  1. Navigate to Jenkins → Project Folder (e.g., CORE-Sandbox-V2) → Project Job (e.g., CORE-Sandbox-V2-PR) → Configure.

  1. In the Build Triggers section, check the option for General Webhook Trigger.

  1. In Token Credential field, select the Webhook Token generated in the previous steps.

  1. In the Pipeline section, select Git repository and SSH credentials generated in the previous steps.
