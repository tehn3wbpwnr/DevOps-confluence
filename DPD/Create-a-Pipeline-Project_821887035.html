DevOps / Pipeline & Deployment : Create a Pipeline Project

Created by Jonathan Bezeau, last modified by Joshua Moore on May 27, 2024

Li’l Test Project

TO-DO convert this to a Pipeline Project

  • Create a “Pipeline” project reflecting the project repository and click Configure

  • Navigate to your new pipeline and click “Configure”

  • Under “Build Triggers,” select “Generic Webhook Trigger” (This requires the Generic Webhook Trigger plugin to be installed if it isn’t already).

    • Create a token for your project:

  • Navigate to your repository on Bitbucket, then select “Respository settings.”

    • On the left hand side, click “Webhooks” and then “Add webhook”

    • Create a title for your webhook. The URL you supply should be in the format: http://JENKINS_URL_HERE/generic-webhook-trigger/invoke?token=YOUR_TOKEN_HERE.

    • Specify the triggers you want for your pipeline and save.

  • Returning to the Jenkins Pipeline configuration page, scroll down to Pipeline and set the drop-down to “Pipeline script from SCM”.

    • Set the SCM to Git

    • Set the repository URL (which you can find on your Bitbucket repo)

    • Set the credentials as the set-up profile (varlab-machine-user)

  • NOTE: UPDATED THESE LATER WHEN PIPELINE IS WORKING Build Steps

    • Create an Execute Windows batch command build step

    • Fill in a Unity command-line build

  • Verify by navigating to your project and clicking “Build Now”

    • Remote to Jenkins server VM and look at logs to determine the cause of any issues

NOTE: If you are setting up a different set of credentials, or you are setting up Jenkins from scratch, you must:

  • Make sure the machine running Jenkins has up-to-date Bitbucket host keys

  • Check that Jenkins is running on a Windows user other than the SYSTEM user.

    • Open Services, locate the Jenkins service, then right click and select Properties.

    • Select the “Log On” tab and check that Jenkins is not running under “Local System account.”

    • If it is, change it to the Windows user you are using for automation.

  • Add new credentials to Jenkins using the SSH private key that you use to connect to Bitbucket, as well as your username (or the username and key you will be using for automation).

Production Projects

With one successful project you can set up others quickly by clicking “+ New Item” on the Dashboard and then entering the name of the thing to copy in the “Copy from” box.
