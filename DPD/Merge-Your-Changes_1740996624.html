DevOps / Pipeline & Deployment : Merge Your Changes

Created by Jonathan Bezeau on Apr 09, 2025

Create a Pull Request

As a CVRI practice, we ask for our changes to be merged to main with a Pull Request. This kicks off our PR Pipeline, which makes sure the branch will not break main when merged, executes all of the testing, performs static analysis, and attaches all of those reports to the Pull Request documentation as an assistance for the developers who will review and approve merge, maybe after some suggested code changes.

Making Requested Changes

Continue working on your local branch to make requested changes, commit and push them, and make your reviewers aware that you have addressed their concerns.

Merge the Pull Request

In CVRI practice, the developer (rather than the reviewer) is the one who clicks Merge on the Pull Request after being told that the request is approved. This will move the ticket status to Done.
