Summary

  • Working with Editmode, Playmode, and WebGL Builds for DLX projects in a Linux headless environment calls for Unity Editors. Without a graphical interface, installing Unity Hub and Editor often proves challenging. Installation becomes feasible by adopting a lightweight desktop environment for graphics and leveraging XRDP to offer user access to the X server. This approach then enables effective use of Editmode, Playmode, and WebGL builds in a headless setup.

1.2 Install a Lightweight Desktop Environment

sudo apt-get update
sudo apt-get install xfce4 xfce4-goodies -y

  • Installing a lightweight desktop environment such as XFCE (XForms Common Environment) on a headless Ubuntu VM allows us to use system resources efficiently while smoothly installing and running graphical applications like Unity Hub and Unity Editor.

About the XFCE

  • Lightweight Desktop Environments for Ubuntu - XFCE

    • https://www.xfce.org/

  • XFCE is a lightweight and highly customizable desktop environment for Unix-like operating systems, including Linux.

    • https://packages.ubuntu.com/search?keywords=xfce4

  • The xfce4-goodies package complements the core XFCE desktop by adding a collection of additional plugins, applications, and utilities.

    • https://packages.ubuntu.com/search?keywords=xfce4-goodies

1.3 Install and Configure a Remote Desktop Protocol server

sudo apt-get update
sudo apt-get install xrdp -y    // Install XRDP
sudo systemctl start xrdp       // Start XRDP Service
sudo systemctl enable xrdp      // Enable XRDP to Start on Boot
echo xfce4-session >~/.xsession // Configure XRDP to Use XFCE Desktop Environment
sudo systemctl restart xrdp     // Restart XRDP Service to Apply Configuration Changes

  • When connecting to a VM via the RDP port (3389) using xrdp, xrdp launches the selected window manager (e.g., Xfce) to start a new X server session. An X server using the X11 protocol is activated in this process, allowing Unity Hub and Unity Editor to be installed and used in GUI mode.

About the XRDP

  • xrdp provides a graphical login to remote machines using RDP (Microsoft Remote Desktop Protocol)

    • https://www.xrdp.org/

    • https://wiki.archlinux.org/title/Xrdp

    • https://learn.microsoft.com/en-us/azure/virtual-machines/linux/use-remote-desktop?tabs=azure-cli

1.4 Configure Firewall Rules and Set Up User Credentials for RDP Access

  • Add the Inbound Port(3389) rule to the networking configuration of the virtual machine

  • Determine User Account and Set Password for RDP Access

whoami  # Displays the current logged-in user's username
sudo passwd your-username  # Changes the password for the specified user

  • Access Ubuntu GUI mode through RDP + public IP address + UserID and password

2.4 Generating SSH key to access Bitbucket repositories

ssh-keygen -t rsa -b 4096 -C "your_email_account"

  • Register the public key(id_rsa.pub) to Bitbucket

    • please follow the instructions as https://varlab-dev.atlassian.net/wiki/spaces/VAR/pages/497123337/Setting+Up+Bitbucket+Sourcetree#Public-Key

  • Setting permission for SSH keys and Directory

# Set read/write permission for the owner only on the private key file (id_rsa)
chmod 600 ~/.ssh/id_rsa
# Set read permission for everyone and read/write for the owner on the public key file (id_rsa.pub)
chmod 644 ~/.ssh/id_rsa.pub
# Set read, write, and execute permissions for the owner only on the .ssh directory
chmod 700 ~/.ssh
# List all files and directories in .ssh with detailed permissions              
ls -l ~/.ssh
# Test SSH key authentication with the Bitbucket server                   
ssh -T git@bitbucket.org
