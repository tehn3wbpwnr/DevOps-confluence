DevOps / Pipeline & Deployment : Automated User Testing

Created by Jonathan Bezeau on Mar 12, 2025

Getting long lists of instructions is time consuming, error prone, and annoying. It is work that some kind of robot should do.

  1. provide a list of instructions that should be completed by a human user

  2. a program performs these steps (or some encoding of these steps) and evaluates after each step if the outcome meets expectations (or some encoding of these expectations)

Opportunities:

  • in Unity editor, build a tool that will interact with the scene and editor commands in real-time

  • as Browser plugin, build a tool that will open the WebGL build attached to a Pull Request, execute the test script, and build a report of steps that meet expectations or don’t

  • Ability to rearrange specified groups of testing steps: if steps A, B, C should be doable in any order to arrive at result D, our tool should be able to randomly order them.

  • Ability to vary accuracy of inputs: when an input is open-ended or should have some acceptable range (numbers, or screen-coordinates) the tool should be able to generate input anywhere within that range to check boundaries and edge conditions. The tool should know if its input is expected to result in success (ability to execute the next step of the plan) or failure (unable to execute next step of the plan)

Other utilities:

  • record testing session using a system tool

  • Unity feature to expose a “bug report” button that saves out the last 30s of gameplay
