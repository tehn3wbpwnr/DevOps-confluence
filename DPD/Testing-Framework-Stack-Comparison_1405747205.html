Summary

This document provides a comparison between the following testing framework stacks, along with a brief summary. Based on my research, I personally recommend Jest + React Testing Library + Supertest based upon the Asset Store’s current stack of (React + Node + Express). However, details are provided below for discussion.

Jest + React Testing Library (for frontend) and Jest + Supertest (for backend)

Pros:

  • Most popular framework; officially supported by React developers and backed by Meta (Facebook), leading to a growing market share and a larger, more recent community with abundant tutorials/resources.

  • More regular updates ensure the framework stays current.

  • Easier setup and configuration allow teams to get up and running quickly.

  • Bundled features include snapshot testing and code coverage, along with built-in UI testing capabilities.

  • Primarily React-based but compatible with Angular, Node, Vue, and other Babel-based projects.

  • Both Have a strong support for CI/CD use, with Jest slightly ahead due to its ability to generate code coverage reports automatically.

Cons:

  • Auto-mocking can potentially slow down runtime, as it automatically mocks all libraries. However, this can often be outweighed by Jest's parallel test execution.

  • Non-React projects may require additional configurations, which can complicate setup.

  • Snapshot testing can create maintenance overhead if not regularly reviewed and updated

Ideal Use Cases:

  • Best suited for modern web applications using React where quick iteration and built-in testing features can significantly streamline the development process.

Mocha + Chai + Enzyme (for frontend) and Supertest + Mocha/Chai (for backend)

Pros:

  • For smaller projects that don’t require sophisticated assertions or testing logic, Mocha is a simple and lightweight solution.

  • Flexible configuration allows teams to choose their preferred assertion library (like Chai) and testing style (TDD vs. BDD), providing tailored testing environments.

Cons:

  • Difficult to auto-mock dependencies, leading to more setup and configuration work.

  • Shrinking market share as alternatives like Jest gain popularity; however, it still maintains a large and dedicated community due to its longevity.

  • Less out-of-the-box support and features compared to Jest.

  • Less frequent updates in recent years may affect long-term support.

Ideal Use Cases:

  • Well-suited for projects requiring flexible configurations and where teams have specific preferences for testing styles or libraries. (Potentially for future vCon projects with a different stack?)

Conclusion

In summary, while both testing frameworks have their strengths and weaknesses, Jest + React Testing Library + Supertest is highly recommended for our current stack due to its modern features, ease of use, and strong community support. It makes a good candidate to try out implementing testing into our CI/CD pipeline with the Asset Store project.

Below are some of the individual points broken up with Notes.

Ease of Setup

Jest + React Testing Library (for frontend) and Jest + Supertest (for backend)

  • Jest is easy to set up with a single command and works out of the box with minimal configuration.

  • React Testing Library is designed to work seamlessly with Jest, making the setup straightforward for React projects.

  • Supertest integrates easily with Jest, allowing for quick testing of Express routes.

Mocha + Chai + Enzyme (for frontend) and Supertest + Mocha/Chai (for backend)

  • Requires more initial configuration (e.g., setting up Enzyme and the appropriate adapter for React).

  • Mocha and Chai provide flexibility but can result in a more complex initial setup.

  • Requires additional setup to run tests in a browser-like environment (e.g., configuring jsdom).

Testing Approach

Jest + React Testing Library (for frontend) and Jest + Supertest (for backend)

  • Unit Testing: Focuses on testing individual components and modules in isolation.

  • User-Centric Testing: React Testing Library promotes testing components from the user's perspective, emphasizing real-world usage.

  • API Testing: Supertest allows for easy testing of HTTP endpoints with clean syntax.

  • Snapshot Testing: Jest supports snapshot testing, enabling you to track changes in your components over time.

Mocha + Chai + Enzyme (for frontend) and Supertest + Mocha/Chai (for backend)

  • Unit Testing: Offers more flexibility and allows for deep testing of components using Enzyme's shallow and mount rendering capabilities.

  • Behavior-Driven Development (BDD): Mocha and Chai encourage a BDD approach, which can be more descriptive and user-friendly for writing tests.

  • API Testing: Supertest can be combined with Mocha/Chai for testing Express routes, although it may require more setup compared to Jest.

Features

Jest + React Testing Library (for frontend) and Jest + Supertest (for backend)

  • Mocking: Built-in mocking capabilities in Jest simplify the process of isolating components and modules.

  • Coverage Reports: Jest includes coverage reporting out of the box, making it easy to track code coverage.

  • Parallel Testing: Jest runs tests in parallel by default, improving performance for large test suites.

  • Easy Integration with CI/CD: Works seamlessly with CI/CD pipelines, especially in environments like Jenkins.

Mocha + Chai + Enzyme (for frontend) and Supertest + Mocha/Chai (for backend)

  • Flexibility: Mocha allows for different styles of testing (e.g., TDD or BDD), making it adaptable to various testing needs.

  • Assertion Libraries: Chai provides multiple styles of assertions (should, expect, assert), allowing for more expressive tests.

  • Custom Configuration: Offers more customization options for advanced users who need fine-grained control over the testing environment.

Popularity

Jest + React Testing Library (for frontend) and Jest + Supertest (for backend)

  • More popular, widely adopted within the React community. Lots of Docs + community support.

  • Stronger ecosystem with more plugins + extensions.

Mocha + Chai + Enzyme (for frontend) and Supertest + Mocha/Chai (for backend)

  • Has also been around a long time with large user base

  • development has fallen down in recent years in comparison to React Testing Library

Performance

Jest + React Testing Library (for frontend) and Jest + Supertest (for backend)

  • Generally faster due to parallel test execution.

  • Certain tests such as snapshot tests can sometimes impact performance based on complexity.

Mocha + Chai + Enzyme (for frontend) and Supertest + Mocha/Chai (for backend)

  • Potentially slower due to need to setup the test environment

  • Sequential test execution unless configured for parallel runs w/ plugin.

I have found conflicting information about performance, some places say one is faster, than the next page says the other one is faster. So take this section with less importance.
