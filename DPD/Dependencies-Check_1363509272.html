Goal

  • Identify and address potential security risks in libraries or packages included in the project before they become issues.

Possible Focus Area

Dependency Management

  • Track all dependencies used in the project and search for vulnerabilities based on known vulnerability databases (e.g., National Vulnerability Database, NVD)

Vulnerability Detection

  • Identify and detect packages and libraries with known security issues through vulnerability scanning of dependencies.

  • Detected vulnerabilities are reported based on the CVE (Common Vulnerabilities and Exposures) database, allowing developers to be aware of the issues and apply patches.

Report Generation

  • Generate reports from the analysis, providing a list of vulnerabilities, their severity, and related packages.

  • This information is useful for developers or security teams to track and resolve the issues.
