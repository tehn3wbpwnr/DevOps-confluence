1. What Jenkins pipeline does for Web developer?

: The primary function of Jenkins pipelines is to verify PR conditions to determine whether it is safe to merge and deploy through various tests.

Breakdown of Pipeline Tasks:

  1. Prepare Workspace: Checks for merge conflicts and prepares the working directory.

  2. Linting Test: Detects lint rule violations in the PR's source code.

  3. Run Unit Tests (Vitest and Jest): Executes client-side and server-side unit tests, and generates code coverage and test reports.

  4. Static Code Analysis: Scans the source code for issues related to security, maintainability, and readability.

Each test result and code coverage rate will be delivered to the corresponding PR page on Bitbucket, allowing PR reviewers to assess the PR with reliable reports.

2. Bitbucket PR Reports Overview

PR-Pipeline Build Status

  • It refers to the pull request pipeline build status.

  • The PR pipeline is triggered when a pull request is created.

  • It has three possible statuses: “SUCCESSFUL”, “FAILED”, and “IN-PROGRESS”.

  • The build number links developers to the corresponding Jenkins build web page associated with the PR.

Test Reports

  • This report includes test results for both client and server projects.

  • Annotations are provided for any test failures.

  • The report title has a link to the lcov code coverage HTML report.

Deployment Pipeline Build Status

  • It refers to the deployment pipeline build status.

  • The deployment pipeline is triggered when a pull request is merged.

  • If the deployment build passed, both deployed client and server versions of container image will be provided with a “SUCCESSFUL” message.

3.1 PR Pipeline

: The status will always be "SUCCESS" or "IN PROGRESS" as long as the Jenkins system is functioning properly.
>> If the status is "FAILED," it indicates one or more build stage failed during the Jenkins build process.

Currently, “lint” and “static code analysis” failure won’t be marked as “FAILED”, but this condition could be updated in the future.

3.2 Deployment Pipeline

: The deployment pipeline is strictly enforced to prevent pushing any container image to Azure Container Registry.
>> Jenkins will return "FAILED," If there is any test failure on the PR branch.

3.3 Jenkins Build Overview Link

: Jenkins provides a link that directs developers to the overview page of each PR or deployment pipeline. This allows developers to track pipeline progress, review test stage results, and access detailed error logs for specific pipeline stages.

4. Test and Coverage Report

: Jenkins provides a test summary report with code coverage rates on the Bitbucket PR page, allowing developers to check for test failures in their working branch and fix any issues before merging.

This report includes:

  • A link to the lcov Code Coverage HTML Report

  • Client or Server Test Summary (failed test cases / total test cases)

  • Code Coverage Rates

  • A List of Failed Tests with descriptions
