DevOps / Pipeline & Deployment : Linux with Unity Headless Environment and Tool Integration

Created by Jindo Kim on Dec 03, 2024

Background

Our current CI/CD pipeline runs on Jenkins with a Windows O/S. In the past, when we searched for information regarding Unity’s compatibility with Linux, we couldn’t find clear or definitive guidance. This uncertainty led us to conclude that Unity’s headless mode was not compatible with the Linux environment.

Moreover, there were previous attempts by Leo Bunting (Unlicensed) and Aaron to migrate our servers to Linux, but unfortunately, They were unsuccessful. This experience reinforced our decision to continue using Windows systems for our pipelines, as the challenges seemed insurmountable at the time.

However, we’ve currently confirmed that we can generate WebGL builds on a Ubuntu headless environment. This opens up the possibility of migrating our pipeline to Linux.

Objectives and Benefits

  • Optimized Resource Utilization: We can achieve better resource management by transitioning to a Linux environment. Linux systems are generally more efficient at handling resources, reducing the likelihood of such issues. This efficiency allows for smoother pipeline executions and minimizes the risk of resource-related disruptions.

  • Enhanced Accessibility: In the Windows environment, access to the VM is limited to RDP (Remote Desktop Protocol), which permits only one DevOps engineer to access. This limitation poses challenges for team collaboration and task organization. It becomes difficult for multiple team members to work simultaneously, which can negatively impact overall team productivity and slow down development processes. On the other hand, Linux environments allow multiple users to access the VM simultaneously via SSH(Secure Shell). This capability enhances accessibility for the entire team, enabling better collaboration, efficient task management, and improved productivity.

  • Improved Scalability: Windows is primarily designed around user login sessions and desktop interactions, which introduces complexities when using it as a server operating system. This design can lead to challenges in automating processes, managing services, and efficiently handling multiple concurrent tasks. These limitations make it difficult to scale our CI/CD infrastructure effectively when using Windows. In contrast, Ubuntu (Linux) is designed to excel in server environments. It provides a robust platform optimized for running services without the overhead of a graphical user interface. Linux systems handle processes and permissions in a way that facilitates automation, efficient resource management, and scalability.

Prerequisites

  • All DLX projects must be able to generate WebGL builds using headless Unity on a Linux environment.

  • Ensure that both Edit Mode and Play Mode executions are functional within the Linux headless setup.
