Create a ticket and a branch

At CVRI we always work in the context of a Jira ticket. In Jira just click Create at the top of the screen, and fill out the information as depicted:

In the CVRI practice for our GitHub flow, we create feature and bug branches in the origin system, Bitbucket, and pull them to our local environment. Branch checkout means that all file changes we work on are staged to that branch. Creating a number of commits, and then push them back to Bitbucket where we do our merge.

The create branch link in Jira will open a new window in Bitbucket that asks you for the repository, branch type (feature or bug) and requires you to set a title. Having your ticket number (VCON-1015 here) in the branch name is critical for keeping the ticket and branch synchronized between the two systems.

When your branch is created, Bitbucket prompts you to check it out. This will set your local environment to focus on an isolated copy of the project.
