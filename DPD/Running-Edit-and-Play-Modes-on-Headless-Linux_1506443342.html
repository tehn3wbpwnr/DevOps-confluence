Summary

  • This document explains enabling Edit Mode and Play Mode in a headless environment after installing Unity Hub and Unity Editor through a Ubuntu GUI (via RDP access). It includes instructions for installing and utilizing X virtual framebuffer(xvfb) to execute the BatchMode command.

1. Environment Preparation

1.1 Install xvfb (X virtual framebuffer) 

sudo apt update
sudo apt install xvfb

  • Installing the xvfb (X Virtual Framebuffer) on a headless Ubuntu VM allows us to run graphical applications like Unity Editor in an environment without a physical display or a fully-fledged X Window Manager. Unlike GUI setups using XFCE and XRDP, which initialize an X Server by default, headless Ubuntu systems accessed via SSH do not start an X Server by design.

    • Although SSH X Server forwarding is an alternative, we have already installed Unity Hub and the DLX Editor using XFCE and XRDP. To streamline tasks like editing, playing, and building DLX projects—including WebGL builds—on a headless setup, we utilize xvfb as a lightweight virtual display solution.

About XVFB

  • Xvfb is an X server that can run on machines with no display hardware and no physical input devices. It emulates a dumb framebuffer using virtual memory.

    • https://www.x.org/archive/X11R7.7/doc/man/man1/Xvfb.1.xhtml#:~:text=Xvfb%20is%20an%20X%20server%20that%20can%20run%20on%20machines%20with%20no%20display%20hardware%20and%20no%20physical%20input%20devices.%20It%20emulates%20a%20dumb%20framebuffer%20using%20virtual%20memory.

2. Editmode Test

2.1 Command:

  • <Editor Path>

  • -batchmode

  • -projectPath "<Project Path>"

  • -testPlatform EditMode

  • -logFile "<Log File Path>"

  • -testResults

  • -debugCodeOptimization

  • -nographics

  • For example, when applied to the respiratory therapy DLX project:

/home/gui/Unity/Hub/Editor/6000.0.12f1/Editor/Unity \
-projectPath "/home/gui/Desktop/respiratory-therapy" \
-batchmode \
-testPlatform EditMode \
-runTests \
-logFile "/home/gui/Desktop/respiratory-therapy/reports/test_results/PlayMode-tests.log" \
-testResults "/home/gui/Desktop/respiratory-therapy/reports/test_results/PlayMode-results.xml" \
-debugCodeOptimization

3. Playmode Test

3.1 Command:

  • -projectPath "<Project Path>"
