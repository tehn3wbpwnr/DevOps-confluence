DevOps / Pipeline & Deployment : Connect using Visual Studio Code

Created by Jonathan Bezeau, last modified on Apr 14, 2025

Using the Remote Explorer (SSH) extension in Visual Studio we can open folders on servers and edit file contents in a more user friendly tool than the LINUX command line. Click on the Remote Explorer icon (shown at the bottom left of the screen capture) and then the SSH settings gear icon in the middle, to open your local the .ssh/config file. Define Host webdlx the way it’s set up in the screenshot:

Host webdlx
  HostName localhost
  IdentityFile ~/.ssh/vconkey.pem
  User vconadmin
  Port 1056

Of course connecting to localhost on port 1056 isn’t going to get us anywhere unless our SSH tunnel is running, so we still need to do that before connecting. In the VS Code Terminal pane, run:

ssh -i ~/.ssh/vconkey.pem vconadmin@vconestoga.com -NL 1056:webdlx:22

Now connect to webdlx in a new window so you don’t kill the running SSH tunnel process by throwing away everything in your current VS Code session.

Your new window will ask you (the first time you connect) what operating system webdlx is using. Click Linux and then open the explorer to go poking around in our home directory!
