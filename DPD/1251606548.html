Terminology

  • LTI: Standard specification allows learning management systems (LMS) or platforms to integrate

  • LTI js Server: A server that allows external access and execution of the DLX

Summary

This document describes the process and structure of synchronizing the LTI js Server between the main branch of the Bitbucket Repository and the running instance of the VM (dlx-web hosting). This can also be considered as the CD (Continous Deployment) pipeline for the LTI js Server.

Configuration Details

Bitbucket Repository Setting

  • URL:

    • Base URL: https://webdlx.vconestoga.com

      • Configured in the nginx configuration to direct traffic to the server.

    • Endpoint(Path): /litwebhook

      • Used in the nginx configuration as a location block to specify where to direct the traffic.

      • Used in the LTI js Server Updater to handle POST requests in the Node.js application.

  • Secret:

    • Secret token

      • Bitbucket Cloud will use your secret token to create a HMAC signature, which will be sent to you as the value of the X-Hub-Signature header with each payload.

      • This token is used in the ‘Data Integrity’ verification step in LTI js Server Updater to ensure the authenticity and integrity of the incoming webhook requests.

  • Trigger Setting

    • Merged option checked

      • Allows the Bitbucket Repository to send a POST request to the registered URL (endpoint) only after a completed merge.

Feather VM

Ngnix Configuration

  • server_name webdlx.vconestoga.com;

    • Specifies the domain name that the server will respond to.

  • # Allow only Bitbucket IP ranges
allow 104.192.136.0/21;
allow 185.166.140.0/22;
allow 13.200.41.128/25;
deny all;

    • Specifies the IP addresses to allow

      • https://support.atlassian.com/bitbucket-cloud/docs/what-are-the-bitbucket-cloud-ip-addresses-i-should-use-to-configure-my-corporate-firewall/ 

  • #Method check
if ($request_method != POST) {
    return 405;
}

    • Specify the method to allow

  • location /ltiwebhook { proxy_pass http://10.5.0.8:4000; }

    • Specifies how to handle requests to the /ltiwebhook path, it is proxied to http://10.5.0.8:4000

Web DLX VM

LTI js Server Updater

  • Location: /home/vconadmin/DLX-Server/LTI_JS_Server_Updater

  • Installed packages

    • express.js

      • Is utilized to configure middleware that processes incoming request bodies as JSON.

    • node-cache

      • Used to cache data (signature) locally within the server to improve performance and reduce the need for database queries.

    • dotenv

  • const PORT = 4000;

    • The server is configured to listen on port 4000.

  • app.post('/ltiwebhook', (req, res) => {}

    • To handle POST requests to the `/ltiwebhook` endpoint.
