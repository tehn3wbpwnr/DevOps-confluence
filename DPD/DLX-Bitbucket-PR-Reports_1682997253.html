1. What Jenkins pipeline does for DLX developer?

: The primary function of Jenkins pipelines is to verify PR conditions to determine whether it is safe to merge and deploy through various tests.

Breakdown of Pipeline Tasks:

  1. Prepare Workspace: Checks for merge conflicts.

  2. Linting Test: Identifies lint rule violations in the PR source code.

  3. Edit Mode Test: Runs edit mode tests under the assumption that the PR branch has been merged into the main branch.

  4. Play Mode Test: Runs play mode tests under the same assumption as the edit mode test.

  5. WebGL Building: Checks for errors or warnings during the WebGL build process.

Each test result or build result is delivered to the corresponding PR page on Bitbucket, allowing PR reviewers to assess the PR with reliable reports.

2. Bitbucket PR Reports Overview

PR-Pipeline Build Status

  • It refers to the pull request pipeline build status.

  • The PR pipeline is triggered when a pull request is created.

  • It has three possible statuses: SUCCESSFUL, FAILED, and IN-PROGRESS.

  • The build number links developers to the corresponding Jenkins build web page associated with the PR.

Deployment Pipeline Build Status

  • It refers to the deployment pipeline build status.

  • The deployment pipeline is triggered when a pull request is merged.

Consolidated Test Report

  • This report includes test results for both edit and play modes.

  • Annotations are provided for any test failures.

  • The report title has a link to the code coverage HTML report.

WebGL Build Report

  • This report refers to the WebGL build result of the corresponding PR branch.

  • If the build is successful, a playable WebGL build link will be provided.

  • If the build fails, the error message, exit code, and full build log file will be included.

Linting Report

  • This report provides details on lint rule violations in the source code.

  • If any rule violations are found, error messages along with the corresponding source code paths will be included as annotations.

3. Build Status

: This build status refers to the Jenkins pipeline build status. However, the PR pipeline and deployment pipeline behave slightly differently.

3.1 PR Pipeline

: The status will always be "SUCCESS" or "IN PROGRESS" as long as the Jenkins system is functioning properly.
>> If the status is "FAILED," it indicates one or more build stage failed during the Jenkins build process.

Currently, lint failure won’t be marked as “FAILED”, but this condition could be updated in the future.

3.2 Deployment Pipeline

: The deployment pipeline is strictly enforced to prevent delivering any WebGL build with errors or test failures.
>> Jenkins will return "FAILED," If there is any test or build failure on the PR branch.

3.3 Build Link

: Jenkins provides a link that directs developers to the overview page of each PR or deployment pipeline. This allows developers to track pipeline progress, review test stage results, and access detailed error logs for specific pipeline stages.

4. Consolidated Test Report

: This is a consolidated test report that combines the results of the edit mode test, play mode test, and code coverage analysis.

This report includes:

  • A link to the Code Coverage HTML Report

  • Edit and Play Mode Test Summary (failed test cases / total test cases)

  • Line Coverage Rate

  • A List of Failed Tests with descriptions

5. Build Report

: The Jenkins pipeline builds WebGL in every PR pipeline run and provides a WebGL HTML link to developers. However, the reporting features differ depending on whether the WebGL build is successful or not.

5.1 WebGL Build Success Report

  • WebGL build HTML link will be provided on the report if project is successfully built.

  • This WebGL build is hosting by dlx-webhost server.

  • System error message may be provided.

This WebGL build is available only while the PR is OPEN. Once the PR is MERGED, the build will be deleted.

This WebGL build is accessible only through the link provided in the Bitbucket “Build Report”.
Any attempt to access the link outside of Bitbucket will be blocked.

5.2 WebGL Build Failure Report

  • Build Result and Unity Batch Mode Exit Code will be provided.

  • System Error Messages will be displayed.

    • Full descriptions are available when hovering over the Error Description field.

  • Unity Error Codes and Descriptions will be included.

  • Bitbucket Source Code Path will be provided.

    • Developers can navigate directly to the error source code.

6. Linting Report

: The Jenkins pipeline checks for lint violations in PR branch source codes based on predefined linting rules. If any violations are detected, they will be reported in the “Linting Report” with an error description and the corresponding source code path.

When developers click the Source Code Link, they are directed to the exact location of the lint error in the Bitbucket repository.

This allows developers to fix errors before merging the PR, ensuring code consistency.
