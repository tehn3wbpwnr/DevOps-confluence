DevOps / Pipeline & Deployment : How to Manually Register The External Tool on eConestoga

Created by Chris Park, last modified by Jonathan Bezeau on Apr 16, 2025

: To host DLX WebGL on eConestoga, we must first register our external tool on the platform. There are two methods for registering an external tool on LMS (eConestoga or Moodle): 'Dynamic Registration' and 'Manual Registration.' This document will focus solely on 'Manual Registration.'

  • 1. Prepare Tool URLs
  • 2. Register External Tool on eConestoga
  • 3. Get Registration Details
  • 4. Insert The Platform 'Registration Details' to MongoDB
  • Video Demonstrations

1. Prepare Tool URLs

: To manually register an external tool on LMS, we need to enter all Tool URLs ourselves. All Tool URLs are made up of a combination of a "Default URL" and an associated endpoint.
>> e.g “https://webdlx.vconestoga.com” + “/endpoint“ = “https://webdlx.vconestoga.com/endpoint”

The essential Tool URLs are as described below:

  • Tool URL: Default URL which links to LTIjs service
    >> e.g. https://webdlx.vconestoga.com
    >> e.g. https://brightspacedlx.vconestoga.com

  • Public keyset: Public key endpoints' URL that external tool supports
    >> e.g. https://webdlx.vconestoga.com/keys
    >> e.g. https://brightspacedlx.vconestoga.com/keys or /jwks

  • Login URL: Login endpoint URL that external tool supports. It will be used after the registration.
    >> e.g. https://webdlx.vconestoga.com/login
    >> e.g.https://brightspacedlx.vconestoga.com/login

  • Redirection URI(s): Redirection URI list that allows access to LMS from the external side.

2. Register External Tool on eConestoga

: In order to actually register an external tool, you need an eConestoga administrator account.

D2L Tool Registration Document: https://community.d2l.com/brightspace/kb/articles/23662-tool-registration-deployment-and-links

  • If you get admin account, go to [ “menu” >> “Manage Extensibility“ ] like the image below

  • Then selects [ “LTI Advantage“ >> “Register Tool” ]

  • Enter all Tool URLs into Standard Registration Configuration Setting

3. Get Registration Details

: If the external tool is successfully registered on eConestoga, we can retrieve all the 'Registration Details' needed to insert into the local MongoDB. A sample of the 'Registration Details' is shown in the image below.

4. Insert The Platform 'Registration Details' to MongoDB

: If we use ‘Dynamic Registration,’ the ‘Registration Details’ will be automatically inserted into the database. However, with the 'Manual Registration' process, the database needs to be updated manually.

LTI.js already provides methods (e.g., registerPlatform) to insert data into MongoDB. You can find the relevant code in index.js located in the ‘devopsdlxserver’ directory. A sample code is provided below.

Click here to expand...
// Setup function
const setup = async () => {
  await lti.deploy({ port: process.env.PORT });

  /**
   * Register platform
   */

  await lti.registerPlatform({
    url: "",
    name: "eConestoga",
    clientId: "",
    authenticationEndpoint:
      "",
    accesstokenEndpoint: "",
    authConfig: {
      method: "JWK_SET",
      key: "",
    },
  });

};


Once all the values are filled out, restart the service to update the new ‘Registration Details’ in MongoDB.
>> mongosh command: db.platforms.find().pretty()


These information will be used in ‘login' process to check the client’s authentication.
