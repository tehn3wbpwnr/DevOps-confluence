DevOps / Pipeline & Deployment : Timeouts

Created by Karandeep Sandhu on Apr 07, 2025

What is Timeout? Basically how long before Jenkins will abort the Pipeline run.

Jenkin allows two ways to set timeouts for builds

  1. Global Timeout via Jenkins Dashboard

  2. Through code within Jenkin files

Jenkins Dashboard #1

Following are steps to enable timeout via Jenkins dashboard

  1. Go to Dashboard → Manage Jenkins → System

  2. Scroll down till you find Global Build Time Out

  3. Enable via the checkbox

  4. Now able to set up a timeout globally for all builds

The Timeout Strategy sets what the condition will be for the pipeline to stop running.

  • Elastic

    • Terminate a build based on a specified percentage of the mean of the duration of the n most recent non-failing builds

  • Absolute

    • Terminate a build based on a fixed time-out period.

  • Deadline

    • Terminate a build based on a deadline time specified in HH:MM:SS or HH:MM (24-hour time format).

  • Likely stuck

    • Terminate a build when it has taken many times longer than previous ones.

  • No Activity

    • Terminate a build when there has not been any log output since the last n seconds.

Code Way #2

Following are steps to enable timeout via code in Jenkin files

  1. Select your Jenkin pipeline file you want to add timeout to

  2. Paste the following code inside the pipeline brace block ( pipeline { } )

    options {
    timeout(time: 1, unit: 'HOURS')
}

  3. Now timeout of 1 hour will be set for that pipeline

Limitation

  • Have to set this timeout to each Jenkins pipeline file you have.

  • Time out strategy is not available when using this option
