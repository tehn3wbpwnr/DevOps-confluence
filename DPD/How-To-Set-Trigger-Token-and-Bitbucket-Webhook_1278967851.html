How To Set Bitbucket Webhook and Jenkins Trigger Token

1. Webhook (Bitbucket)

: Bitbucket Webhook allows us to send the build trigger signal to Jenkins in certain trigger conditions such as create, update, or merge pull request.
So, we can start the Jenkins build in the situation we want to start the test, build, or deploy the project in the Bitbucket repository.
This step will explain how to create and manage Webhooks in Bitucket.

** To manage Webhook, the Bitbucket account should have proper permission on the target repository **

  • Click [Add webhook] if you want to create a new webhook

  • Click [Edit] if you want to manage existing webhook

>> You can use anything you want for the trigger token, but just being aware of this token SHOULD BE MATHCED with the secret of 'trigger-token' in Jenkins.

2. Trigger Token (Jenkins)

: Trigger Token is used to trigger the Jenkins build by webhook from the specific Bitbucket repository.
So, the ‘secret text’ of this ‘trigger-token’ should be matched to the Bitbucket webhook setting so that Jenkins can recognize webhooks and trigger builds correctly.

>> Add the Credential likewise explained in How To Set Bitbucket App Password, 2. Create Credential in Jenkins.

>> Make sure the name of the credential should be ‘trigger-token’

>> Make sure the secret text should be matched with the Webhooks’ trigger token
