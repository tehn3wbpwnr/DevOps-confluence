DevOps / Pipeline & Deployment : Connect to our webdlx server

Created by Jonathan Bezeau, last modified on Apr 14, 2025

We’re going to go through our entry point server, to tunnel a connection to the server where we host our builds. First you need to open a command prompt and establish a local port-forward connection:

ssh -i .ssh/vconkey.pem vconadmin@vconestoga.com -NL 1056:webdlx:22

Leave the first command prompt running. What it’s doing is sending every bit of data it sees on port 1056 to vconestoga.com and from there to webdlx on the normal SSH port (22). This way we don’t have to leave webdlx' SSH port open to the public internet.

In another command prompt, we’re going to connect to our own computer on port 1056… the port we defined as our local forward:

ssh -i .ssh/vconkey.pem vconadmin@localhost -p 1056
