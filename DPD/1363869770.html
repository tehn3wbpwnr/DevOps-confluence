Goal

  • To identify issues or bugs that arise when various components (modules/units) interact with each other.

Focus Areas

UI Rendering and Interaction Testing

  • Ensures that modules or components are rendered correctly and that they interact as expected.

  • Verifies that UI updates and states are generated correctly when specific events occur.

  • Examples:

    • Render React components with shallow or full DOM rendering and test interactions between components with Enzyme npm.

    • Simulate specific events on components to test their behavior with Enzyme’s simulate.

  • Jest Snapshot Testing: A typical snapshot test case renders a UI component, takes a snapshot, then compares it to a reference snapshot file stored alongside the test.

State Management, Data Flow, and Mocking

  • Mocks external dependencies to achieve consistent test results without relying on external systems, such as API calls, database connections.

  • Tests state management logic, data flow, and UI updates in response to state changes.

  • Examples

    • For testing state management logic (reducers, actions, asynchronous actions) and React component state changes with Jest

    • Mocking and spying on functions to ensure correct operation with Sinon or MSW(Mock Service Worker)

      • https://www.npmjs.com/package/sinon - Package Health Score

      • https://www.npmjs.com/package/msw - Package Health Score

Validating Outputs and Side Effects

  • Verify that the expected outputs are returned correctly and that any side effects (e.g., data storage, logging) are handled properly.

  • Examples:

    • Check if specific values are returned as expected and verify if side effects occurred with Jest' Matchers

    • Spy on functions to ensure they were called the correct number of times and that side effects were processed correctly with Sinon npm

Operations

Create an integration testing plan.

  1. Establish the testing strategy and decide which modules or components to integrate and test.

  2. Design test cases and scenarios.

  3. Tool or Package related: Jira, Confluence, Test Case Management

Create test data, test cases, and use cases

  1. Generate diverse input data that might occur in a real operating environment.

  2. Code each test case.

    1. Node.js - [Integration Testing with Node JS and Jest - Medium](https://salithachathuranga94.medium.com/integration-testing-with-node-js-and-jest-8df71d65ca2a )

      1. API Layer — perform real API calls through code and assert

      2. DAO Layer — manipulate some test data with DAO methods and assert

      3. Service Layer — perform real service calls and assert

    2. React

      1. [Increase Code Maintainability With React Integration Testing](https://www.toptal.com/react/react-testing-library-tutorial)

      2. https://www.npmjs.com/package/enzyme - [Package Health Score] created by AirBnB

        1. Using Enzyme with Jest

        2. What Is the Difference Between Jest and Enzyme?

Prepare the test environment. (if this step is necessary)

  1. Set up an environment where tests can be performed in local, staging, or CI/CD environments.

  2. Mock or configure the state of external services like databases or servers.

  3. Tools:

    1. Docker: Set up a consistent testing environment using container-based solutions

Run test cases after unit integration.

  1. Execute the written test cases.

  2. Tools:

    1. Jest (used the link provide previously)

    2. Enzyme (used the link provide previously)

    3. https://www.npmjs.com/package/supertest - [Package Health Score] for testing HTTP

Detect integration errors.

  1. After executing the tests, analyze the logs to identify where errors occurred, and use debugging tools to trace the location of the problem.

    1. It is necessary to invest in debugging tools considering the language, project libraries, testing framework, and libraries used for testing

Apply required fixes to errors.

Retest functionality after fixing bugs.

Repeat the testing cycle until all errors are fixed.
