DevOps / Pipeline & Deployment : Creating The Pipeline On Jenkins

Created by Jonathan Bezeau on Apr 09, 2025

The first step will be setting up a new folder for your project. From the main Jenkins dashboard you can select “New Item”

Here we make a new folder, as at the highest level (The main page) we use folders to contain our different projects.

On the empty folder click the “+ New Item” again or the “Create New Job” in the middle of the screen to get the following screen.

Once the pipeline has been created we want to then start configuring it.

And we can start selecting certain settings, they will be shown below.

Next we need to create a trigger token, this is attached to the trigger(Webhook).

This trigger token gets used when we hook up the Webhook, so remember it!

Now at the bottom we setup where we get the Jenkins File from

Additionally we provide the link to our repository in the “Repository URL” section you can get that here: https://bitbucket.org/VARLab/devops-pipeline/src/main/ Select clone then copy the text as shown below

Next we need to tell the pipeline where in the repository the file is located. At the very bottom there is “Script Path”

Now we can save our changes! The next step will be setting up a Webhook on BitBucket, to do this you will need access to an account with admin credentials on a Repository such as our CI-Catherine account, please contact a team member for the login information.
